## 提案內容概要

### 🎯 目標
通過實施縮圖機制和漸進式載入技術，將照片牆載入時間減少 50% 以上，提升用戶體驗。

### 🔧 主要變更
1. **縮圖生成**：上傳時自動生成 150px 寬度縮圖
2. **漸進式載入**：點擊照片時先顯示縮圖，背景載入原圖
3. **向後相容**：舊照片繼續正常顯示，使用原圖作為後備
4. **遷移機制**：提供批量遷移工具為現有照片生成縮圖

### 📋 規格變更
- **新增需求**：縮圖生成和存儲、漸進式載入、照片牆縮圖顯示、向後相容性、照片遷移支援
- **修改需求**：照片上傳流程包含縮圖生成、照片列表 API 返回縮圖信息

### 🏗️ 技術決策
- 使用 Sharp 庫進行服務端圖片處理
- 縮圖尺寸設定為 150px 寬度（適合手機三列布局）
- 實現漸進式載入而非純懶加載
- 採用可選遷移策略，降低部署風險
