## 介面互動流程

### 1. 檔案大小顯示流程

```mermaid
sequenceDiagram
    participant U as 用戶
    participant UI as 介面
    participant API as 後端API
    participant DB as 資料庫
    
    U->>UI: 開啟照片管理頁面
    UI->>API: 請求照片列表
    API->>DB: 查詢照片資料（含檔案大小）
    DB-->>API: 返回照片列表
    API-->>UI: 返回照片資料
    UI->>UI: 格式化檔案大小顯示
    UI-->>U: 顯示照片列表（含檔案大小）
```

### 2. 照片詳情查看流程

```mermaid
sequenceDiagram
    participant U as 用戶
    participant UI as 介面
    participant API as 後端API
    
    U->>UI: 點擊照片卡片
    UI->>UI: 顯示詳情彈窗
    UI->>UI: 顯示檔案大小等詳細資訊
    UI-->>U: 展示完整照片資訊
```

## 視覺設計規範

### 1. 檔案大小顯示格式
- **小於 1 MB**: 顯示為 "XXX KB" (如: 800 KB)
- **1 MB - 1024 MB**: 顯示為 "X.X MB" (如: 2.5 MB)
- **大於 1024 MB**: 顯示為 "X.X GB" (如: 1.2 GB)

### 2. 顏色編碼（可選）
- **小檔案 (< 1 MB)**: 綠色 🟢
- **中檔案 (1-5 MB)**: 藍色 🔵
- **大檔案 (5-10 MB)**: 橙色 🟠
- **超大檔案 (> 10 MB)**: 紅色 🔴

### 3. 圖示使用
- **檔案大小**: 📁 或 📊
- **用戶**: 👤
- **愛心/投票**: ❤️
- **時間**: 🕒
- **公開/隱私**: 👁 / 👁‍🗨

## 響應式設計考量

### 1. 桌面版
- 照片網格: 5 列顯示
- 完整統計資訊顯示
- 詳細的檔案大小資訊

### 2. 平板版
- 照片網格: 3-4 列顯示
- 簡化統計資訊
- 保持檔案大小顯示

### 3. 手機版
- 照片網格: 2 列顯示
- 基本統計資訊
- 檔案大小以簡潔格式顯示

這個設計提供了完整的檔案大小顯示功能，同時考慮了使用者體驗和視覺層次。
