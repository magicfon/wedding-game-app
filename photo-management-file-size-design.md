## 設計特點

### 1. 視覺元素
- **檔案大小圖示**：使用 📁 圖示表示檔案大小
- **簡潔格式**：使用 "2.5 MB", "800 KB" 等簡潔格式
- **顏色編碼**：可選擇性地使用顏色區分檔案大小範圍

### 2. 位置安排
- **照片列表**：在每張照片卡片底部顯示檔案大小
- **詳情彈窗**：在照片資訊區域突出顯示檔案大小
- **統計區域**：新增總大小和平均大小統計

### 3. 互動功能
- **排序功能**：可按檔案大小排序照片
- **篩選功能**：可按檔案大小範圍篩選照片
- **統計圖表**：顯示檔案大小分布圖表

## 技術實現要點

### 1. 資料庫
- 新增 `file_size` 欄位到 `photos` 表
- 儲存檔案大小（位元組）

### 2. 後端 API
- 更新照片列表 API 返回檔案大小
- 新增檔案大小統計 API

### 3. 前端組件
- 使用現有的 `formatFileSize` 函數格式化顯示
- 在適當位置顯示檔案大小資訊
- 新增統計圖表組件

## 使用者體驗考量

1. **資訊層次**：檔案大小作為次要資訊，不影響主要功能
2. **視覺平衡**：確保新增的檔案大小資訊不會讓介面顯得擁擠
3. **實用性**：提供統計和篩選功能，幫助管理員管理儲存空間

這個設計提供了完整的檔案大小顯示功能，同時保持了介面的簡潔性和易用性。
